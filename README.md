# トコトコ駅歩き（京都観光×健康ウォーキング）

歩数・GPS距離を自動計測し、京都市営地下鉄「東西線」の駅間距離に換算して表示するWebアプリです。
消費カロリーの食べ物換算、適正水分量、区間沿線の観光地情報、マスコットのアニメーションなどを備えています。

## 技術構成
- バックエンド: Python (Flask)。ルーティングのみを担当し、計算ロジックは `calc.py`(歩数/カロリー/水分/食べ物換算) と `stations.py`(駅換算) に分離
- フロントエンド: HTML / CSS / JavaScript(ESモジュール、フレームワーク不使用)
- 地図: Leaflet + OpenStreetMap（CDN経由）
- データ保存: ブラウザの `localStorage`（サーバー側にDBなし）

## ローカルでの起動方法
```bash
cd eki-hokei
pip install -r requirements.txt
python app.py
```
`http://127.0.0.1:5000` を開いてください。スマホでの確認は、ブラウザの検証ツールでスマホ表示に切り替えるか、同一Wi-Fi内のスマホから `http://(PCのIPアドレス):5000` にアクセスしてください。
**GPS機能はブラウザがHTTPS(またはlocalhost)である場合のみ動作します**。スマホ実機で試す場合は、後述のデプロイ後にHTTPSのURLでアクセスしてください。

## 無料での公開方法（例）
Render / PythonAnywhere / Railway など、Python(Flask)の無料枠があるサービスにそのままデプロイできます。`app.py` と `requirements.txt` をアップロードし、起動コマンドを `python app.py`（もしくは `gunicorn app:app`）に設定してください。

## 実装済み機能
### 自動計測（このアプリは手入力に対応していません）
- ページを開くと、加速度センサー(DeviceMotion API)とGPS(Geolocation API)を**自動で起動**します。ON/OFFを切り替える専用スイッチはありません
- iOSのSafariは「センサー利用許可」をユーザー操作の直後にしかリクエストできない仕様のため、自動起動が失敗した場合は画面への最初のタップを合図に自動で再試行します(専用ボタンは表示されません)
- GPSは実際に歩いた経路を地図に描画します
- **重要な制限**: どちらも、この画面(タブ)を開いている間だけ計測できます。ブラウザ/アプリを閉じている間の計測はWeb技術だけでは実現できません（実現するにはCapacitor等でのネイティブアプリ化が必要です）

### 画像について
マスコット・食べ物アイコン・プロフィールアイコンは `static/img/` 配下の `<img>` タグとして実装しており、現在は仮のプレースホルダー画像が入っています。正式なイラストができ次第、同じファイル名で差し替えるだけで反映されます(`mascot.png`, `food_icon.png`, `profile_placeholder.png`)。

### 駅換算・観光
- 東西線17駅(六地蔵〜太秦天神川)の駅データに対応。駅番号・ローマ字・ひらがなも収録し、実際の駅名標(駅サイン)を模したデザインで表示。烏丸線データも同梱(アカウント画面から切替可能)
- 現在の区間(例：「T10 東山」→「T09 蹴上」)を駅名標風カードで表示
- 区間を歩き切ると、到着をお祝いするアニメーション(マスコットのジャンプ+紙吹雪)
- 目的駅周辺の観光地情報を表示(Googleマップ検索へのリンク付き)
- Leaflet地図で、現在の区間とGPS実測ルートを表示

### 健康サポート
- 体重が分かればMET法でより正確にカロリーを計算(不明な場合は歩数からの概算にフォールバック)
- 消費カロリーを身近な食べ物に換算(例:「🍙 おにぎり 1.5個分」)
- 歩行時間から適正水分補給量の目安を表示(例:「150ml：コップ1.5杯」)

### 継続しやすい仕組み
- アチーブタブ: 路線図スタンプ(通過した駅を線でつないで表示)、累計距離チャレンジ(京都→大阪/東京など)、日別の記録一覧。**このタブのデザインは今後見直し予定**で、現状は簡易表示です
- アカウントタブ: 身長・体重・目標歩数・起点駅/路線/方向の設定。将来のログイン機能のプレースホルダー付き

## データについて（重要）
`data/stations.json` について:
- 駅順・総距離・沿線の主な観光地は公開情報に基づく実データです
- **各駅間の区間距離、および各駅の緯度経度は地図表示用の概算値**です。大会提出前に、京都市交通局の公式資料や地図サービスで正確な値を確認し、差し替えることを推奨します
- 観光地情報の外部リンクはGoogleマップ検索へのリンクとして自動生成しており、施設の公式サイトへのリンクではありません

## ファイル構成
```
eki-hokei/
├── app.py                  # Flaskエントリーポイント(ルーティングのみ)
├── calc.py                 # 歩数・カロリー・食べ物・水分の計算ロジック
├── stations.py              # 駅換算ロジック
├── requirements.txt
├── data/
│   └── stations.json       # 路線・駅・観光地データ
├── templates/
│   └── index.html
└── static/
    ├── css/style.css
    └── js/
        ├── main.js          # 全体の初期化・画面更新の司令塔
        ├── state.js         # localStorageでの状態管理
        ├── api.js           # サーバーAPI呼び出し
        ├── ruler.js         # 路線図スタンプ(アチーブタブ)描画
        ├── route_card.js    # 現在区間(駅→駅)の表示
        ├── mascot.js        # マスコットの描画・アニメーション状態管理
        ├── map.js            # Leaflet地図の描画
        ├── pedometer.js     # 加速度センサーによる歩数検出
        ├── gps_tracker.js   # GPSによる距離計測
        └── tabs.js          # 下部タブ切り替え
```
